# Задание 3.2: Алгоритм создания пользователя на стороне бэк-сервиса

## Пошаговый алгоритм

```
[Начало]
   │
   ▼
1. ПОЛУЧЕНИЕ ЗАПРОСА
   │ Бэк-сервис получает POST /auth/register
   │ с JSON-телом запроса
   │
   ▼
2. ПАРСИНГ И ДЕСЕРИАЛИЗАЦИЯ
   │ Парсинг JSON-тела в объект DTO
   │ Если парсинг не удался → ответ 400 (INVALID_JSON)
   │
   ▼
3. ВАЛИДАЦИЯ ФОРМАТА (по полям)
   │
   ├─ 3.1 email: проверка формата (regex / regexp)
   │    Если невалиден → ошибки[{field: "email", message: "Некорректный формат"}]
   │
   ├─ 3.2 password: длина >= 8 и <= 128
   │    Если невалиден → ошибки
   │
   ├─ 3.3 confirmPassword == password
   │    Если не совпадает → ошибки[{field: "confirmPassword", message: "Пароли не совпадают"}]
   │
   ├─ 3.4 firstName: непустая строка, maxLength 100
   │    Если невалиден → ошибки
   │
   ├─ 3.5 lastName: непустая строка, maxLength 100
   │    Если невалиден → ошибки
   │
   ├─ 3.6 phone (если указан): regex /^\+7\d{10}$/
   │    Если невалиден → ошибки
   │
   └─ 3.7 agreeTerms: boolean, должен быть true
        Если false → ошибки[{field: "agreeTerms", message: "Требуется согласие"}]
   │
   ▼
4. ПРОВЕРКА ОБЯЗАТЕЛЬНЫХ ПОЛЕЙ
   │ Если хотя бы одно обязательное поле отсутствует → 400 (VALIDATION_ERROR)
   │ Если есть ошибки валидации → 400 с массивом ошибок
   │
   ▼
5. ПРОВЕРКА УНИКАЛЬНОСТИ EMAIL
   │ Запрос в БД: SELECT id FROM users WHERE email = ?
   │ Если пользователь найден → 409 (USER_ALREADY_EXISTS)
   │
   ▼
6. ХЭШИРОВАНИЕ ПАРОЛЯ
   │ Генерация соли (bcrypt cost factor = 12)
   │ Хэширование пароля: hash = bcrypt.hash(password, salt)
   │ Пароль в открытом виде НЕ сохраняется
   │
   ▼
7. СОЗДАНИЕ ЗАПИСИ В БД
   │ INSERT INTO users (
   │   id,              -- UUID v4 (usr_xxxx)
   │   email,
   │   password_hash,
   │   first_name,
   │   last_name,
   │   phone,           -- NULL если не указан
   │   agree_terms,     -- true
   │   status,          -- 'active'
   │   created_at,      -- now()
   │   updated_at       -- now()
   │ )
   │ Если ошибка БД → 500 (INTERNAL_SERVER_ERROR)
   │
   ▼
8. ОТПРАВКА ПИСЬМА ПОДТВЕРЖДЕНИЯ (async)
   │ Генерация токена подтверждения
   │ Отправка email с ссылкой подтверждения
   │ (не блокирует ответ пользователю)
   │
   ▼
9. ФОРМИРОВАНИЕ ОТВЕТА
   │ Формирование RegisterSuccessResponse:
   │ {
   │   "success": true,
   │   "data": { id, email, firstName, lastName, phone, createdAt }
   │ }
   │ Пароль и password_hash НЕ включаются в ответ
   │
   ▼
10. ОТПРАВКА ОТВЕТА
    │ HTTP 201 Created
    │ Content-Type: application/json
    │
    ▼
[Конец]
```

## Обработка ошибок

| Код | Тип ошибки | Описание |
|-----|-----------|----------|
| 400 | VALIDATION_ERROR | Ошибки валидации полей |
| 400 | INVALID_JSON | Невалидный JSON в теле запроса |
| 409 | USER_ALREADY_EXISTS | Email уже зарегистрирован |
| 422 | UNPROCESSABLE_ENTITY | Невалидный формат данных |
| 429 | RATE_LIMIT_EXCEEDED | Превышен лимит запросов |
| 500 | INTERNAL_SERVER_ERROR | Внутренняя ошибка сервера |

## Диаграмма активности (описание)

```
              ┌─────────────┐
              │   Получение  │
              │   запроса    │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │   Парсинг    │──Невалидный──▶ 400 INVALID_JSON
              │   JSON       │
              └──────┬──────┘
                     │ OK
              ┌──────▼──────┐
              │  Валидация   │──Ошибки────▶ 400 VALIDATION_ERROR
              │  полей       │
              └──────┬──────┘
                     │ OK
              ┌──────▼──────┐
              │  Проверка    │──Найден────▶ 409 USER_ALREADY_EXISTS
              │  уникальности│
              └──────┬──────┘
                     │ Не найден
              ┌──────▼──────┐
              │  Хэширование │
              │  пароля      │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │  Запись в    │──Ошибка────▶ 500 INTERNAL_SERVER_ERROR
              │  БД          │
              └──────┬──────┘
                     │ OK
              ┌──────▼──────┐
              │  Отправка    │ (async, не блокирует)
              │  email       │
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │  Формирование│
              │  ответа 201  │
              └─────────────┘
```
