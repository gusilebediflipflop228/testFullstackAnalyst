# Задача 2: Публикация товара на маркетплейсе

## User Story

> Как **продавец**, я хочу **опубликовать свой товар на маркетплейсе (витрине товаров)**, чтобы **увеличить охват аудитории и получить дополнительные продажи**.

### Критерии приёмки

- Продавец может создать карточку товара с обязательными полями (название, описание, цена, фотографии)
- Продавец может выбрать категорию товара из справочника
- Продавец может указать характеристики (размер, цвет, вес и т.д.)
- Продавец может установить количество на складе
- После публикации товар отображается на витрине маркетплейса
- Продавец может предварительно просмотреть карточку перед публикацией
- Продавец может отредактировать опубликованный товар
- Продавец может снять товар с публикации (скрыть с витрины)

## Use Case: UC-01 Создание карточки товара

| Поле | Описание |
|------|----------|
| **Название** | UC-01: Создание карточки товара |
| **Актёры** | Продавец |
| **Предусловие** | Продавец авторизован в личном кабинете |
| **Постусловие** | Карточта товара создана в статусе «Черновик» |
| **Приоритет** | Высокий |

**Основной сценарий:**
1. Продавец переходит в раздел «Мои товары»
2. Продавец нажимает «Добавить товар»
3. Система отображает форму создания товара
4. Продавец заполняет обязательные поля:
   - Название товара
   - Описание
   - Цена
   - Категория (из справочника)
   - хотя бы 1 фотография
5. Продавец заполняет необязательные поля (характеристики, вес, размеры)
6. Продавец указывает количество на складе
7. Продавец нажимает «Сохранить как черновик»
8. Система валидирует данные
9. Система сохраняет товар в статусе «Черновик»
10. Система показывает уведомление об успешном создании

**Альтернативные сценарии:**
- 4а. Обязательные поля не заполнены → система подсвечивает ошибки
- 4б. Название превышает лимит символов → система показывает ошибку
- 4ц. Цена указана с ошибкой (отрицательная) → система показывает ошибку
- 8а. Файл фотографии превышает лимит → система показывает ошибку «Файл слишком большой»

## Use Case: UC-02 Публикация товара на витрину

| Поле | Описание |
|------|----------|
| **Название** | UC-02: Публикация товара на витрину |
| **Актёры** | Продавец |
| **Предусловие** | Товар создан в статусе «Черновик» |
| **Постусловие** | Товар опубликован и виден на витрине маркетплейса |
| **Приоритет** | Высокий |

**Основной сценарий:**
1. Продавец открывает карточку товара (черновик)
2. Продавец нажимает «Опубликовать»
3. Система проводит финальную валидацию всех обязательных полей
4. Система проверяет соответствие товара правилам маркетплейса
5. Система меняет статус товара на «Опубликован»
6. Система индексирует товар для поиска на витрине
7. Система показывает подтверждение «Товар опубликован»
8. Товар доступен на витрине маркетплейса

**Альтернативные сценарии:**
- 3а. Обнаружены ошибки валидации → система показывает список ошибок, публикация отменяется
- 4а. Товар не соответствует правилам (запрещённая категория, запрещённый контент) → система блокирует публикацию с объяснением причины

## Use Case: UC-03 Редактирование опубликованного товара

| Поле | Описание |
|------|----------|
| **Название** | UC-03: Редактирование опубликованного товара |
| **Актёры** | Продавец |
| **Предусловие** | Товар опубликован на витрине |
| **Постусловие** | Данные товара обновлены |
| **Приоритет** | Средний |

**Основной сценарий:**
1. Продавец открывает карточку опубликованного товара
2. Продавец нажимает «Редактировать»
3. Система отображает форму редактирования с текущими данными
4. Продавец вносит изменения
5. Продавец нажимает «Сохранить»
6. Система валидирует данные
7. Система сохраняет изменения
8. Система обновляет индекс поиска (если изменились ключевые поля)
9. Система показывает подтверждение

**Альтернативные сценарии:**
- 6а. Валидация не пройдена → система показывает ошибки

## Use Case: UC-04 Снятие товара с публикации

| Поле | Описание |
|------|----------|
| **Название** | UC-04: Снятие товара с публикации |
| **Актёры** | Продавец |
| **Предусловие** | Товар опубликован на витрине |
| **Постусловие** | Товар скрыт с витрины, статус изменён на «Снят с публикации» |
| **Приоритет** | Средний |

**Основной сценарий:**
1. Продавец открывает карточку товара
2. Продавец нажимает «Снять с публикации»
3. Система запрашивает подтверждение
4. Продавец подтверждает действие
5. Система меняет статус товара на «Снят с публикации»
6. Система удаляет товар из индекса поиска витрины
7. Система показывает подтверждение

**Альтернативные сценарии:**
- 4а. Продавец отменяет → система закрывает диалог без изменений

## Use Case: UC-05 Предварительный просмотр карточки товара

| Поле | Описание |
|------|----------|
| **Название** | UC-05: Предварительный просмотр |
| **Актёры** | Продавец |
| **Предусловие** | Карточка товара заполнена (черновик или редактируется) |
| **Постусловие** | Продавец видит как будет выглядеть карточка на витрине |
| **Приоритет** | Низкий |

**Основной сценарий:**
1. Продавец нажимает «Предпросмотр»
2. Система отображает карточку товара в формате витрины
3. Продавец просматривает название, описание, фото, цену
4. Продавец закрывает предпросмотр
5. Система возвращает к форме редактирования
